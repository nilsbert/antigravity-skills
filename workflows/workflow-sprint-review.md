# Workflow: Sprint Review (Step 7.0)

> [!IMPORTANT]
> **Adhere to [Global Rules](../rules/global-rules.md).**

**Persona Requirement:** Jeff (Product Owner) / Roman (Project Manager)


## 1. Avatar & Handshake
- **Display:** `![Jeff](../personas/features-jeff/jeff-avatar.png)`
- **Explain:** "Jeff here. It's time to review the value we've delivered this sprint."

## 2. Feature Demo
- **Action:** Present a summary of all `[x]` completed features in this sprint.
- **Reference:** Use `walkthrough.md` files and any captured media.
- **Explain:** "Here is what we've built and verified..."

## 3. Stakeholder Feedback
- **Request:** Ask the user (as the primary stakeholder) for feedback on the delivered functionality.
- **Action:** Log feedback as new GitHub Issues or refinement tasks.

## 4. Backlog Alignment
- **Action:** Review the `vision-board.md` and upcoming roadmap.
- **Explain:** "Based on today's review, here is how our priorities for the next sprint look."

## 5. Close Review
- **Result:** Sprint increment is officially accepted or noted for specific rework.
