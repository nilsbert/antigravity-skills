# Workflow: User Story Mapping (3.2)

> [!IMPORTANT]
> **Adhere to [Global Rules](../rules/global-rules.md).**
> We transform the journey into a logical "Backbone" and "Walking Skeleton".

**Persona Requirement:** [Jeff](../personas/features-jeff/jeff-avatar.png)

## 1. Defining the Backbone
1. **Source:** Read the User Journey Map from `docs/product/journeys/`.
2. **Translate Action:** Convert "Steps" from the journey into "Broad Activities" (The Backbone).
3. **Table Layout:** Use `../templates/story-map.md`.

## 2. Populating Tasks
- **Action:** For each Activity, list the specific User Tasks underneath it.
- **Rule:** Tasks must be granular enough to be estimated.

## 3. Slicing for Releases
1. **Slice 1 (Walking Skeleton)**: What is the absolute minimum we need to perform the full end-to-end flow? (The MVP).
2. **Slice 2 (V1.0)**: What adds enough value to be "Marketable"?
3. **The Line:** Define the "MVP Line" clearly in the table.

## 4. Documentation
1. **Create File**: Save to `docs/product/story-maps/[name].md`.
2. **Link Features**: Link existing or new `feat-XXX` IDs to the tasks in the map.

## 5. Next Step
- **Decision:** Once the map is prioritized, proceed to `/workflow-refinement-1-discovery`.
