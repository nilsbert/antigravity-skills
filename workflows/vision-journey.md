# Workflow: User Journey Mapping (3.1)

> [!IMPORTANT]
> **Adhere to [Global Rules](../rules/global-rules.md).**
> This workflow bridges the gap between static Personas and actionable Story Maps.

**Persona Requirement:** [Roman](../personas/product-management-roman/roman-avatar.png)

## 1. Journey Discovery
1. **Identify Persona**: Select a persona from `docs/product/personas/`.
2. **Define Scope**: What is the start and end of this specific journey? (e.g., "From hearing about a tender to clicking 'Relay to Teams'").

## 2. Emotional Analysis
- **Action:** Ask about the user's emotional state at each stage (Awareness, Research, Action, Completion).
- **Goal:** Identify the "Pain Points" where the user is most likely to churn or fail.

## 3. Documentation
1. **Create File**: Save the map to `docs/product/journeys/[name].md`.
2. **Template**: Use `../templates/user-journey-map.md`.
3. **Roman's Sign-off**: "Does this journey actually solve the 'Needs' defined in the Vision Board?"

## 4. Next Step
- **Decision:** If the journey is complete, trigger `/workflow-vision-story-mapping`.
- **🛑 STOP:** If the journey contradicts the Vision, return to `/vision-publish`.
